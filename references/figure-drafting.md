# 说明书附图规范（patent-drafting · 附图详细规则）

> 调用方：`references/spec-writing.md` §11（仅留硬规则与路由，详规则在此）。
> 闭环：选什么图 → 图内画到什么程度 → 生成合规 A4 Word → 自动+人工验收。
> 生成走 `tools/patent_figures.py`，验收走 `tools/patent_figures_check.py`（命令见末尾）。

## 一、选图决策表（三类案件的稳定组合，规律非铁律）

| 案件类型 | 核心附图组合 |
|---|---|
| 方法及系统 | 总体系统结构、改进前后结构差异、关键循环/判断流程、（可选）验证曲线 |
| 人工智能网络 | 方法流程、网络总体数据流、创新模块细图、训练关系、（可选）结果对比 |
| 通信方法 | 通信场景/系统拓扑、信号/帧结构、总体流程、优化子流程及终止判断、（可选）性能曲线 |

按案件实际裁剪：缺少创新模块就不画模块细图；无可靠实验就不画结果图；不机械凑图数。

## 二、共同原则

- **一图一抽象层级**：摘要级/系统级/数据流级/流程级/结构级/关系级各自独立；不要把系统关系、数据流、网络推理、训练监督塞进同一张图。
- **结果图不替代结构图与流程图**：三者各司其职。
- **训练 / 推理 / 系统运行不混成一条流程**：训练是离线一次性、推理是在线每次、系统运行是部署态；混在一张流程里会被理解为"每次推理都重训"。

## 三、线型语义（强约束）

- **实线**：普通前向数据流、网络层间连接、**跳跃连接**。
- **虚线**：**仅训练阶段**的数据/梯度/监督连接；须配**图例**标注"虚线为仅训练阶段"。
- 跳跃连接**不得**与训练监督共用虚线（否则语义冲突，审查易误解）。
- 一张图内不要混用多义虚线；训练关系单独成图或单独子图。

## 四、流程图粒度

- 总体流程与子流程分开：总体流程给步骤主链，子流程（如迭代/优化/终止判断）单独出图。
- 迭代/优化类须画出固定量、更新量、收敛判据或最大次数、输出（与正文"迭代与终止步骤"对应）。
- 离线训练与在线推理用**两条泳道/两个子图**分离，并用一条"部署（仅一次）"连接表明训练后模型如何进入推理。

## 五、子图规则

- 同一图号下用 **图N(a) / 图N(b)**；适合"同一对象的两个视角"（如网络的推理结构 vs 训练关系）。
- 子图须成对/成组出现（a、b、…连续），不得有裸主号与子号并存的歧义。
- 正文与附图说明对子图的引用一致（"如图N(a)所示"）。

## 六、网络结构图画法

- 编解码/U 型：输入 → 逐级下采样编码侧 → 最深层 → 逐级上采样解码侧 → 输出；显式画各级、跳跃连接（实线）、输入输出。
- 训练关系单独出**图N(b)**：训练样本、网络预测、监督目标（含真值→标签/笔画图）、各损失、损失聚合、反向传播与参数更新、训练后模型；全部用虚线 + 图例。
- 标准固定主干、发明点不在网络本身时，推理结构用框图级即可，不必画张量立方体；有新颖网络模块才加模块细图（C×H×W 维度标注 + 操作块 + 符号图例）。

## 七、结果图规则

- 仅在有可靠实验条件时加；实施例暂缓或无实验则**不加**（贸然加会引入证据与图文联动风险）。
- 加则标注坐标轴含义、对比对象、指标口径，且数据须可追溯。

## 八、摘要附图

- 指定一张作摘要附图（`is_abstract:true`），只保留最高层技术链（输入→核心处理→训练后模型→输出）。
- 缩至约 4×6cm 仍清晰：用竖版、少框、短词；不堆系统/网络/训练细节。

## 九、版面合规（由 `patent_figures.py` 固定，勿手工拼）

- A4 210×297mm；页边距 上25/下15/左25/右15mm；**无页眉**（官方只要图号+页码+必要文字）。
- 页脚居中 PAGE 页码域；图片与"图N(±a)"居中；"图片+图号"为不可拆单元、默认一图一页（小图可同页，由配置 `same_page_group`）。
- 附图页**只排"图N(±a)"**，完整图名仅进说明书"附图说明"。
- 图片中性元数据；灰度/低饱和；有效 DPI ≥150（细长/简单图按内嵌显示尺寸计，不按裸像素）。

## 十、逐图联动表（防错引，须逐项核对）

| 图号 | 子图号 | 配置 full_name | 附图说明声明 | 正文（具体实施方式）引用 |
|---|---|---|---|---|
| 图N | a/b/无 | 与附图说明一致 | "图N为……" | "如图N所示/从图N可以看出" |

- 新增/改/删任一图，四列同步；主图号连续无缺号；子图成对。
- 显式防参考稿常见错误：图N 说明错引图M 子图、正文分析图N 却写成"由图M 可见"、引用了不存在的图号。
- 这些可由 `patent_figures_check.py` 自动核对（图文双向引用、连续图号、子图成对、无悬空引用）。

## 十一、两层验收

- **自动硬检查**（`patent_figures_check.py`）：A4、页脚 PAGE、图片数=配置、图片/题注居中、题注=有序"图N(±a)"无长名、连续图号+子图成对、无残留 mermaid/代码、灰度/低饱和、有效 DPI、图文双向引用、无悬空图号。
- **人工视觉门禁**：把各图按 2/3 缩放与 4×6cm 渲染后**目视**确认文字、箭头、图例清晰、灰度可辨——此项不能纯自动判定。

## 十二、命令

```bash
# 生成（配置→严格渲染→A4 DOCX）
conda run --no-capture-output -n patent-drafting python "$HOME/.claude/skills/patent-drafting/tools/patent_figures.py" \
  --config <figures.json> --out-docx <5-说明书附图.docx> --assets-dir <附图>
# 验收（不改文件；说明书传 .docx 或 .md/.txt，.doc 先转换）
conda run --no-capture-output -n patent-drafting python "$HOME/.claude/skills/patent-drafting/tools/patent_figures_check.py" \
  --config <figures.json> --figures-docx <5-说明书附图.docx> --spec <说明书.md|.docx>
```

配置 JSON 每图字段：`number, subfigure(null|a|b), type, abstraction, source({mmd:路径}|{image:路径}), full_name, is_abstract, orientation, line_legend[{style,meaning}], spec_section, same_page_group`。长 mermaid 放各自 `.mmd`，不塞 JSON。
